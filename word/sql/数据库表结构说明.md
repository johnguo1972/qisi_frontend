## 1. 用户与认证模块

### 1.1 user_account（用户账号表）

**表名**：`user_account`  
**中文名**：用户账号表  
**作用**：存储所有用户的基本信息和角色，支持多角色体系（教师/学生/家长/管理员）

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 用户ID | BigAutoField | 主键，自增 |
| role_type | 用户角色 | VARCHAR(20) | teacher/student/parent/admin |
| login_name | 登录名 | VARCHAR(64) | 可选登录名 |
| mobile | 手机号 | VARCHAR(20) | 唯一，作为用户唯一标识 |
| display_name | 显示名称 | VARCHAR(64) | 用户昵称/姓名 |
| subject | 教师学科 | VARCHAR(20) | 仅教师角色使用 |
| stages | 教师学段 | JSONField | 教师教授的学段列表，如['小学', '初中', '高中'] |
| avatar_url | 头像URL | VARCHAR(255) | 用户头像链接 |
| status | 用户状态 | VARCHAR(20) | active/inactive/banned |
| created_at | 创建时间 | TIMESTAMP | 用户注册时间 |
| updated_at | 更新时间 | TIMESTAMP | 信息最后更新时间 |

**关键索引**：mobile（唯一索引）

---

### 1.2 student_parent_bind（学生家长绑定表）

**表名**：`student_parent_bind`  
**中文名**：学生家长绑定表  
**作用**：建立学生与家长的关联关系，支持一个学生绑定多个家长

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 绑定ID | BigAutoField | 主键 |
| student_user_id | 学生用户 | ForeignKey | 关联用户表，学生端 |
| parent_user_id | 家长用户 | ForeignKey | 关联用户表，家长端 |
| relation_type | 关系类型 | VARCHAR(20) | father/mother/guardian |
| bind_status | 绑定状态 | VARCHAR(20) | pending/active/disabled |
| bound_at | 绑定时间 | TIMESTAMP | 绑定生效时间 |

**关键约束**：唯一约束（student_user_id + parent_user_id）

## 2. 机构与班级管理模块

### 2.1 institution（教育机构表）

**表名**：`institution`  
**中文名**：教育机构表  
**作用**：存储学校、培训机构等教育机构信息，支持多租户架构

| 字段名 | 中文名 | 类型 | 说明 |

| contact_name | 联系人姓名 | VARCHAR(100) | 机构联系人 |
| contact_phone | 联系电话 | VARCHAR(20) | 机构联系电话 |
| contact_email | 联系邮箱 | VARCHAR(200) | 机构邮箱 |
| address | 机构地址 | VARCHAR(500) | 详细地址 |

### 2.2 institution_member（机构成员表）

**表名**：`institution_member`  
**中文名**：机构成员表  
**作用**：关联用户与机构，定义用户在机构中的角色（管理员/教师）

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 成员ID | BigAutoField | 主键 |
| institution | 所属机构 | ForeignKey | 关联机构表 |
| user | 用户 | ForeignKey | 关联用户表 |

**关键约束**：唯一约束（institution + user）

### 2.3 class（班级表）

**表名**：`class`  
**中文名**：班级表  
**作用**：存储班级基本信息，每个班级属于一个机构，由教师管理

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 班级ID | BigAutoField | 主键 |
| class_no | 班级编号 | VARCHAR(20) | 自动生成，如CLS-12345678，唯一 |
| institution | 所属机构 | ForeignKey | 关联机构表 |
| creator_teacher | 创建教师 | ForeignKey | 创建该班级的教师 |
| class_name | 班级名称 | VARCHAR(200) | 如"高一数学实验班" |
| description | 班级描述 | TEXT | 班级简介 |
| max_students | 最大人数 | INTEGER | 班级容量上限，默认50 |
| invite_code | 邀请码 | VARCHAR(8) | 学生加入班级的邀请码，唯一 |
| allow_invite_join | 允许邀请加入 | BOOLEAN | 是否允许学生通过邀请码加入 |
| status | 班级状态 | VARCHAR(20) | active/disabled |
| created_at | 创建时间 | TIMESTAMP | 班级创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 信息更新时间 |

**关键索引**：class_no（唯一）、invite_code（唯一）

---

### 2.4 class_teacher（班级教师表）

**表名**：`class_teacher`  
**中文名**：班级教师表  
**作用**：关联教师与班级，定义教师在班级中的角色（班主任/协作教师）

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 关系ID | BigAutoField | 主键 |
| class_obj | 所属班级 | ForeignKey | 关联班级表 |
| teacher | 教师 | ForeignKey | 关联用户表（教师角色） |
| role | 教师角色 | VARCHAR(20) | owner/co_teacher |

**关键约束**：唯一约束（class_obj + teacher）

---

### 2.5 class_student（班级学生表）

**表名**：`class_student`  
**中文名**：班级学生表  
**作用**：关联学生与班级，记录学生加入班级的方式和状态

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 关系ID | BigAutoField | 主键 |
| class_obj | 所属班级 | ForeignKey | 关联班级表 |
| student | 学生 | ForeignKey | 关联用户表（学生角色） |
| join_type | 加入方式 | VARCHAR(20) | invite/manual/import |
| status | 学生状态 | VARCHAR(20) | active/removed/pending |
| joined_at | 加入时间 | TIMESTAMP | 学生加入班级的时间 |

**关键约束**：唯一约束（class_obj + student）

---

### 2.6 class_join_request（班级加入申请表）

**表名**：`class_join_request`  
**中文名**：班级加入申请表  
**作用**：记录学生申请加入班级的请求，支持审批流程

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 申请ID | BigAutoField | 主键 |
| class_obj | 目标班级 | ForeignKey | 关联班级表 |
| applicant | 申请人 | ForeignKey | 关联用户表 |
| applicant_name | 申请姓名 | VARCHAR(100) | 申请时填写的姓名 |
| applicant_phone | 申请电话 | VARCHAR(20) | 申请时填写的电话 |
| request_type | 申请类型 | VARCHAR(20) | invite_code/self_apply |
| status | 审批状态 | VARCHAR(20) | pending/approved/rejected |
| message | 申请说明 | TEXT | 学生申请时的备注信息 |
| handled_by | 处理人 | ForeignKey | 审批该申请的教师 |
| handled_at | 处理时间 | TIMESTAMP | 审批完成的时间 |
| created_at | 创建时间 | TIMESTAMP | 申请提交的时间 |

## 3. 试卷管理模块

### 3.1 tiku_exam_paper（试卷表）

**表名**：`tiku_exam_paper`  
**中文名**：试卷表  
**作用**：存储教师上传的试卷信息，包括Word/PDF文件和解析状态

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 试卷ID | BigAutoField | 主键 |
| title | 试卷名称 | VARCHAR(255) | 如"2025高考数学全国卷" |
| paper_code | 试卷编号 | VARCHAR(20) | 自动生成，如M90001，唯一 |
| region | 来源地区 | VARCHAR(100) | 全国卷/广东卷/深圳卷 |
| subject | 学科 | VARCHAR(50) | 数学/物理/化学 |
| stage | 学段 | VARCHAR(50) | 小学/初中/高中 |
| grade | 年级 | VARCHAR(50) | 高一/高二/高三 |
| paper_type | 试卷类型 | VARCHAR(50) | 月考/期中/期末/高考真题 |
| has_solution | 含答案解析 | BOOLEAN | 是否包含标准答案和解析 |
| uploaded_by | 上传者 | ForeignKey | 上传试卷的教师 |
| source_file_path | 源文件路径 | VARCHAR(500) | Word/PDF原始文件路径 |
| pdf_file_path | PDF路径 | VARCHAR(500) | 转换后的PDF文件路径 |
| total_pages | 总页数 | INTEGER | PDF总页数 |
| total_questions | 总题数 | INTEGER | 解析出的题目总数 |
| status | 试卷状态 | VARCHAR(50) | uploaded/converting/parsing/finished/failed |
| error_message | 错误信息 | TEXT | 解析失败时的错误详情 |
| is_deleted | 已删除 | BOOLEAN | 软删除标记 |
| created_at | 创建时间 | TIMESTAMP | 上传时间 |
| updated_at | 更新时间 | TIMESTAMP | 最后更新时间 |

**关键索引**：paper_code（唯一）、is_deleted

---

### 3.2 tiku_parse_task（解析任务表）

**表名**：`tiku_parse_task`  
**中文名**：解析任务表  
**作用**：记录试卷解析的异步任务，支持11步AI解析流水线

| 字段名 | 中文名 | 类型 | 说明 |
|---|---|---|---|
| id | 任务ID | BigAutoField | 主键 |
| paper | 所属试卷 | ForeignKey | 关联试卷表 |
| question | 目标题目 | ForeignKey | 关联题目表（单题重解析时使用） |
| task_type | 任务类型 | VARCHAR(50) | full_parse/single_reparse |
| status | 任务状态 | VARCHAR(50) | pending/running/success/failed/cancelled |
| progress | 进度 | INTEGER | 0-100百分比 |
| current_step | 当前步骤 | VARCHAR(255) | 如"第3步：AI定位题目区域" |
| error_message | 错误信息 | TEXT | 任务失败原因 |
| retry_count | 重试次数 | INTEGER | 已重试次数 |
| celery_task_id | Celery任务ID | VARCHAR(255) | Celery异步任务的UUID |
| started_at | 开始时间 | TIMESTAMP | 任务启动时间 |
| finished_at | 完成时间 | TIMESTAMP | 任务完成时间 |
| created_at | 创建时间 | TIMESTAMP | 任务创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 任务状态更新时间 |

---

### 3.4 tiku_question_id_counter（题目系统编号计数器）

### 4.3 tiku_exam_question（题目表）⭐核心

**表名**：`tiku_exam_question`

|---|---|---|---|

| paper | 所属试卷 | ForeignKey | 关联试卷表 |

| paper_question_no | 试卷题号 | VARCHAR(50) | 组合编号，如M90001-1-3 |
| parent_question | 父题 | ForeignKey | 关联自身，用于子题关系 |
| section_title | 大题标题 | VARCHAR(255) | 如"一、选择题" |
| question_type | 题型 | VARCHAR(50) | single_choice/multiple_choice/fill_blank等 |
| subject | 学科 | VARCHAR(50) | 数学/物理 |
| stem | 题干 | TEXT | 题目主体文本，含LaTeX公式 |
| stem_html | 题干HTML | TEXT | HTML格式题干 |
| answer | 答案 | TEXT | 标准答案，如"A"、"BC" |
| analysis | 解析 | TEXT | 答案解析说明 |
| solution | 详解 | TEXT | 完整解题步骤 |
| comment | 点评 | TEXT | 教师点评 |
| raw_explanation | 原始解释 | TEXT | OCR提取的解释文本 |
| raw_text | 原始文本 | TEXT | OCR原始文本 |
| knowledge_points | 知识点 | JSONField | 关联的知识点ID列表 |
| difficulty | 难度 | DECIMAL(4,2) | 题目难度0-10 |
| original_question | 原始题目 | ForeignKey | 用于变式题关系 |
| page_start | 起始页 | INTEGER | 题目在PDF中的起始页码 |
| page_end | 结束页 | INTEGER | 题目在PDF中的结束页码 |
| bbox | 主区域bbox | JSONField | 题目在页面中的坐标区域 |
| region_json | 跨页区域 | JSONField | 跨页题目的区域信息 |
| sort_order | 排序 | INTEGER | 题目在试卷中的顺序 |
| confidence | 置信度 | DECIMAL(5,4) | AI解析置信度 |
| formula_need_review | 公式需复核 | BOOLEAN | LaTeX公式是否需人工检查 |
| need_review | 需人工校验 | BOOLEAN | 是否需要教师审核 |
| review_status | 复核状态 | VARCHAR(50) | unreviewed/reviewed/confirmed/rejected |
| parse_status | 解析状态 | VARCHAR(50) | auto_parsed/manual_created |
| ai_answer_a | AI答案A模式 | JSONField | AI生成的标准答案模式 |
| ai_answer_b | AI答案B模式 | JSONField | AI生成的提示式答案模式 |
| ai_answer_c | AI答案C模式 | JSONField | AI生成的苏格拉底式答案模式 |
| ai_knowledge_enrichment | AI知识点enrich | JSONField | AI扩充的知识点信息 |
| ai_probe_result | AI探查结果 | JSONField | 第1步：题目类型探查 |
| ai_vision_extract | AI读图结果 | JSONField | 第2步：图表信息提取 |
| ai_verifier_result | AI校验结果 | JSONField | 第3步：答案合理性校验 |
| ai_processed_at | AI处理时间 | TIMESTAMP | AI流水线完成时间 |
| ai_processing_status | AI处理状态 | VARCHAR(20) | pending/running/success/failed |
| created_at | 创建时间 | TIMESTAMP | 题目创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 题目更新时间 |

**关键索引**：system_id（唯一）、paper_question_no、question_type、review_status
