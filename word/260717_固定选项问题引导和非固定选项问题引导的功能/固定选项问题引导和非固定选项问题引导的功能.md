## 1. 现状分析

### 1.1 当前核心问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **B 模式 options 为空** | ⚠️ 阻塞 | `start_guidance` 读取 `ai_answer_b.get('options')`，但 AI 实际生成的数据结构是 `questions[].options`，顶层无 `options` 键，导致始终返回 `[]` |
| **B 模式无分步引导** | ⚠️ 阻塞 | 当前后端只返回 1 个 hint + 空选项，没有利用 `questions` 数组中的 3-4 个递进问题做分步引导 |
| **C 模式无切换下一题** | ⚠️ 阻塞 | 引导页只处理单题，完成后直接返回，没有"下一题"功能，用户需要退出引导页再进入下一题 |
| **引导页和答题页分离** | ⚠️ 体验差 | 引导页是独立页面，无法看到原题题干和选项，用户需要来回切换 |
| **B 模式无数据兜底** | ⚠️ 体验差 | `ai_answer_b` 为 `None` 时，后端返回空选项和默认提示，没有提供通用引导选项 |
| **无加载动画** | ⚠️ 体验差 | 请求接口时无 loading 状态，用户无法感知网络请求正在执行 |
| **错题本入口无 mode 参数** | ⚠️ 功能缺失 | `wrongbook.vue` 导航到引导页时没传 `mode` 参数，后端需自动推荐 |

### 1.2 当前数据流

```
答题页 (answer.vue)
  ├─ 点击"固定选项问题引导" → 跳转 guidance.vue?mode=B
  └─ 点击"非固定选项问题引导" → 跳转 guidance.vue?mode=C

错题本 (wrongbook.vue)
  └─ 点击错题 → 跳转 guidance.vue（无 mode 参数）

引导页 (guidance.vue)
  ├─ onMounted → startGuidance({ question_id, mode_type })
  │     └─ POST /student/guidance/sessions → 后端返回 { session_id, mode, hint, options }
  └─ 用户选择/输入 → guidanceReply(session_id, reply)
        └─ POST /student/guidance/sessions/{id}/reply → 后端返回 { next_hint, is_completed }
```

## 2. 功能需求总览

### 2.1 核心功能需求

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F-01 | B 模式分步引导 | P0 | 利用 `ai_answer_b.questions` 数组做 3-4 步递进选择题引导 |
| F-02 | B 模式选项展示 | P0 | 每步展示当前问题的 4 个选项（A/B/C/D），用户选择后推进到下一步 |
| F-03 | B 模式步骤反馈 | P0 | 用户选择后显示正确/错误反馈，并显示正确答案和解析 |
| F-04 | B 模式无数据兜底 | P0 | `ai_answer_b` 为 null 时使用通用引导选项 |
| F-05 | C 模式多步推进 | P0 | 利用 `ai_answer_c.questions` 数组做 3-5 步开放问题引导 |
| F-06 | C 模式 LLM 评价 | P0 | 用户回答后调用 LLM 进行评价，然后给出下一个问题 |
| F-07 | C 模式实时降级 | P0 | `ai_answer_c` 为空调用 LLM 实时生成，仍失败则降级 B 模式 |
| F-08 | 引导页内切换下一题 | P0 | 当前题引导完成后，可点击"下一题"进入同一关卡的下一题 |
| F-09 | 引导页展示原题题干 | P1 | 引导页顶部展示原题题干和选项，方便用户对照 |
| F-10 | 引导页展示AI总结 | P1 | 引导完成后展示 `summary` 和 `final_answer` |
| F-11 | 提交答案联动 | P1 | 引导完成后自动提交该题答案（可选），或标记为已引导 |
| F-12 | 加载动画 | P1 | 请求接口时显示 loading 动画 |
| F-13 | 错题本入口自动推荐模式 | P1 | 错题本导航到引导页时，后端根据题型自动推荐 B/C 模式 |

### 2.2 页面交互需求

| ID | 需求 | 描述 |
|----|------|------|
| UI-01 | 引导页布局 | 左侧为对话区/引导区，右侧为操作区 |
| UI-02 | 原题展示 | 引导页顶部展示当前题目的题干和选项（可选） |
| UI-03 | 进度指示器 | B 模式显示当前步骤 (1/3, 2/3, 3/3) |
| UI-04 | 选项按钮 | B 模式选项以大按钮形式展示，点击后高亮 |
| UI-05 | 反馈展示 | 选择后显示正确/错误标记 + 解析 |
| UI-06 | 加载动画 | 请求接口时显示骨架屏或旋转 loading |
| UI-07 | 下一题入口 | 引导完成后显示"下一题"按钮 |
| UI-08 | 返回按钮 | 引导页顶部或底部有"返回做题"按钮 |

## 3. 用户流程设计

### 3.1 B 模式完整流程（固定选项问题引导）

```
用户从答题页点击"固定选项问题引导"
  │
  ▼
导航到引导页，传入 questionId, levelId, mode=B
  │
  ▼
引导页加载
  ├─ 调用 startGuidance({ question_id, mode_type: 'B' })
  │     └─ POST /api/v1/student/guidance/sessions
  │
  ▼
后端 start_guidance 处理：
  ├─ 1. 读取 ai_answer_b.questions[0] 作为第一步
  │     ├─ hint = questions[0].question
  │     ├─ options = questions[0].options → ["A. 内容", "B. 内容", "C. 内容", "D. 内容"]
  │     └─ step_index = 0, total_steps = 3
  ├─ 2. 如果 ai_answer_b 为 null → 使用通用兜底选项
  └─ 3. 创建 AIGuidanceSession，记录 step_index=0
  │
  ▼
引导页展示：
  ├─ 显示原题题干（从 levelDetail 缓存获取）
  ├─ 显示进度 "第 1/3 步"
  ├─ 显示当前引导问题题干（hint）
  └─ 显示 4 个选项按钮 [A][B][C][D]
  │
  ▼
用户点击一个选项（如 B）
  │
  ▼
调用 guidanceReply(session_id, "B")
  └─ POST /api/v1/student/guidance/sessions/{id}/reply
  │
  ▼
后端 guidance_reply 处理：
  ├─ 1. 读取 content_log_json 中的 step_index
  ├─ 2. 从 ai_answer_b.questions[step_index] 获取 correct_option
  ├─ 3. 判断用户答案是否正确
  ├─ 4. 记录答案到 content_log_json.answers
  ├─ 5. step_index += 1
  ├─ 6. 如果 step_index < total_steps:
  │     └─ 返回 { next_question, options, is_correct, correct_answer, analysis, step_index, is_completed: false }
  └─ 7. 如果 step_index >= total_steps:
        └─ 返回 { summary, final_answer, is_completed: true }
  │
  ▼
引导页展示反馈：
  ├─ 显示 ✅/❌ 标记
  ├─ 显示正确答案和解析
  ├─ 0.5 秒后自动推进到下一步（或点击"下一步"按钮）
  └─ 展示下一个引导问题 + 4 个选项
  │
  ▼
（重复步骤直到所有步骤完成）
  │
  ▼
引导完成页：
  ├─ 展示 summary（总结）
  ├─ 展示 final_answer（最终答案）
  ├─ 展示"提交答案"按钮（可选，自动提交该题答案）
  └─ 展示"下一题"按钮 → 切换到同关卡下一题的引导
  └─ 展示"返回做题"按钮 → 返回答题页
```

### 3.2 C 模式完整流程（非固定选项问题引导）

```
用户从答题页点击"非固定选项问题引导"
  │
  ▼
导航到引导页，传入 questionId, levelId, mode=C
  │
  ▼
引导页加载
  ├─ 调用 startGuidance({ question_id, mode_type: 'C' })
  │     └─ POST /api/v1/student/guidance/sessions
  │
  ▼
后端 start_guidance 处理：
  ├─ 1. 读取 ai_answer_c.questions[0].question 作为第一个问题
  ├─ 2. 如果 ai_answer_c 为空 → 调用 LLM 实时生成
  ├─ 3. 如果 LLM 也失败 → 降级到 B 模式
  └─ 4. 创建 AIGuidanceSession，记录 step_index=0
  │
  ▼
引导页展示：
  ├─ 显示原题题干
  ├─ 显示当前引导问题（开放性问题）
  └─ 显示文本输入框 + 发送按钮
  │
  ▼
用户输入思考内容，点击发送
  │
  ▼
调用 guidanceReply(session_id, "用户输入内容")
  │
  ▼
后端 guidance_reply 处理：
  ├─ 1. 读取 step_index
  ├─ 2. 调用 LLM 评价用户回答
  ├─ 3. step_index += 1
  ├─ 4. 如果 step_index < total_steps:
  │     └─ 返回 { evaluation, next_question, step_index, is_completed: false }
  └─ 5. 如果 step_index >= total_steps:
        └─ 返回 { evaluation, summary, final_answer, is_completed: true }
  │
  ▼
引导页展示反馈：
  ├─ 显示 AI 评价
  └─ 显示下一个引导问题（或完成状态）
  │
  ▼
（重复步骤直到所有步骤完成）
  │
  ▼
引导完成页：（同 B 模式）
  ├─ 展示 summary
  ├─ 展示 final_answer
  ├─ "下一题"按钮
  └─ "返回做题"按钮
```

### 3.3 多题切换流程

```
引导完成页 → 用户点击"下一题"
  │
  ▼
引导页检查：
  ├─ 当前关卡是否有下一题？
  ├─ 是 → 调用 startGuidance 启动下一题的引导
  │        ├─ 增加导航参数: nextQuestionId, levelId
  │        └─ 重新初始化引导状态
  └─ 否 → 提示"已经是最后一题"→ 返回答题页
```

### 4.4 B 模式下需要新增的辅助函数

**`guidance_views.py` 新增辅助函数：**
```python
def _build_question_info(q) -> dict:
    """从 ExamQuestion 对象提取题干和选项，用于引导页展示原题。"""
    return {
        'stem': q.stem or '',
        'stem_html': q.stem_html or '',
        'question_type': q.question_type,
        'options': [
            {'label': o.option_label, 'content': o.content}
            for o in q.options.all()
        ],
        'images': [
            {'id': img.id, 'file_path': img.file_path}
            for img in q.images.all()
        ],
    }


def _extract_b_mode_step(ai_b: dict, step_index: int, total_steps: int = 3) -> dict:
    """从 ai_answer_b 中提取指定步骤的引导数据。

    返回: {
        'hint': str,           # 当前步骤的引导问题
        'options': list,       # ["A. 内容", "B. 内容", "C. 内容", "D. 内容"]
        'correct_option': str, # 正确答案标签（兜底时为空字符串，表示无正确性判断）
        'analysis': str,       # 该步骤解析（兜底时为空）
    }
    如果 ai_b 为空或无数据，返回通用兜底数据。
    """
    questions = (ai_b or {}).get('questions') or []
    if questions and step_index < len(questions):
        q = questions[step_index]
        opts = q.get('options') or {}
        return {
            'hint': q.get('question', '请思考下一步'),
            'options': [f"{k}. {v}" for k, v in opts.items() if k in ('A','B','C','D')],
            'correct_option': q.get('correct_option', ''),
            'analysis': q.get('analysis', ''),
        }
    # 兜底：通用引导选项（无 correct_option，表示不判断正确性）
    fallback_steps = [
        {'hint': '请仔细阅读题干，分析题目的已知条件和未知条件', 'options': ['A. 分析已知条件，找出关键信息', 'B. 思考适用的公式或定理', 'C. 尝试代入具体数值进行验证', 'D. 回顾相关知识点，建立解题思路']},
        {'hint': '根据第一步的分析，尝试列出解题所需的公式或步骤', 'options': ['A. 列出所有已知公式', 'B. 确定解题顺序', 'C. 代入数值进行计算', 'D. 验证结果的合理性']},
        {'hint': '最后检查你的答案，确认是否完整且符合题意', 'options': ['A. 答案已完整，可以提交', 'B. 需要补充中间步骤', 'C. 需要重新检查计算', 'D. 需要换个思路']},
    ]
    if step_index < len(fallback_steps):
        return {**fallback_steps[step_index], 'correct_option': '', 'analysis': ''}
    return {'hint': '引导完成', 'options': [], 'correct_option': '', 'analysis': ''}


def _extract_c_mode_step(ai_c: dict, step_index: int) -> dict:
    """从 ai_answer_c 中提取指定步骤的开放引导问题。

    返回: {
        'question': str,  # 引导问题
        'reference_answer': str,  # 参考答案
        'key_points': list,  # 关键点
    }
    """
    questions = (ai_c or {}).get('questions') or []
    if questions and step_index < len(questions):
        q = questions[step_index]
        return {
            'question': q.get('question', '请继续思考'),
            'reference_answer': q.get('reference_answer', ''),
            'key_points': q.get('key_points', []),
        }
    # 兜底
    return {
        'question': '请继续思考这道题的解题思路',
        'reference_answer': '',
        'key_points': [],
    }
```

def start_guidance(request):
    """S-06: 启动引导。"""
    question_id = request.data.get('question_id')
    mode_type = request.data.get('mode_type', '')

    # 1. 获取题目

        q = ExamQuestion.objects.get(pk=question_id)

        return Response({'code': 404, 'message': '题目不存在', 'data': None, 'trace_id': make_trace_id()}, status=404)

    # 2. 自动推荐模式
    if not mode_type:
        if q.question_type in ('single_choice', 'multiple_choice'):
            mode_type = 'B'
        else:
            mode_type = 'C'

    # 3. 清理过期 session
    AIGuidanceSession.objects.filter(
        student_user_id=request.user, session_status='running',
        created_at__lt=timezone.now() - timedelta(hours=24)
    ).update(session_status='expired')

    ai_b = _as_dict(q.ai_answer_b)
    ai_c = _as_dict(q.ai_answer_c)

    # 4. 创建 session
    session = AIGuidanceSession.objects.create(
        student_user_id=request.user, question_id=question_id,
        mode_type=mode_type, session_status='running',
        content_log_json={'step_index': 0, 'steps': [], 'answers': []},
    )

    # 5. B 模式处理
    if mode_type == 'B':
        questions = (ai_b or {}).get('questions') or []
        total_steps = len(questions) or 3  # 动态计算：AI 数据有就用 AI 的，否则固定 3 步兜底
        step_data = _extract_b_mode_step(ai_b, 0, total_steps)
        is_fallback = not questions
        return Response({'code': 0, 'data': {
            'session_id': session.id, 'mode': 'B',
            'step_index': 0, 'total_steps': total_steps,
            'hint': step_data['hint'], 'options': step_data['options'],
            'is_fallback': is_fallback,
            'question_info': _build_question_info(q),
        }})

    # 6. C 模式处理
    questions = (ai_c or {}).get('questions') or []
    total_steps = len(questions) or 3

    if questions:
        # 优先使用 ai_answer_c 的预置数据
        step_data = _extract_c_mode_step(ai_c, 0)
        return Response({'code': 0, 'data': {
            'session_id': session.id, 'mode': 'C',
            'step_index': 0, 'total_steps': total_steps,
            'hint': step_data['question'],
            'question_info': _build_question_info(q),
        }})

    # ai_answer_c 为空：实时调用 LLM 生成
    try:
        generated = call_qwen_for_guidance_with_question(q.stem, q.answer)
        if generated and generated.get('steps'):
            ai_c = generated
            total_steps = len(generated.get('steps', []))
            first_q = generated['steps'][0]['question']
            return Response({'code': 0, 'data': {
                'session_id': session.id, 'mode': 'C',
                'step_index': 0, 'total_steps': total_steps,
                'hint': first_q,
                'question_info': _build_question_info(q),
            }})
    except Exception:
        pass

    # LLM 也失败：降级到 B 模式
    session.mode_type = 'B'
    session.session_status = 'downgraded'
    session.save()
    questions_b = (ai_b or {}).get('questions') or []
    total_steps_b = len(questions_b) or 3
    step_data = _extract_b_mode_step(ai_b, 0, total_steps_b)

        'session_id': session.id, 'mode': 'B',
        'step_index': 0, 'total_steps': total_steps_b,

        'downgrade_reason': '非固定选项引导数据不可用，已降级到固定选项引导模式',
        'question_info': _build_question_info(q),
    }})

@api_view(['POST'])
@permission_classes([IsAuthenticated, IsStudent])
def guidance_reply(request, session_id):
    """S-07: 推进引导。"""
    try:
        session = AIGuidanceSession.objects.get(pk=session_id, student_user_id=request.user)
    except AIGuidanceSession.DoesNotExist:
        return Response({'code': 404, 'message': '引导会话不存在', 'data': None, 'trace_id': make_trace_id()}, status=404)
    if session.session_status != 'running':
        return Response({'code': 4001, 'message': '引导已结束', 'data': None, 'trace_id': make_trace_id()}, status=400)

    user_reply = (request.data.get('reply') or '').strip()
    try:
        q = ExamQuestion.objects.get(pk=session.question_id)
    except ExamQuestion.DoesNotExist:
        return Response({'code': 404, 'message': '题目不存在', 'data': None, 'trace_id': make_trace_id()}, status=404)

    ai_b = _as_dict(q.ai_answer_b)
    ai_c = _as_dict(q.ai_answer_c)

    log = session.content_log_json or {'step_index': 0, 'steps': [], 'answers': []}
    step_index = log.get('step_index', 0)

    # C→B 降级（连续无效输入）
    if session.mode_type == 'C' and _is_invalid_reply(user_reply):
        session.invalid_input_count += 1
        if session.invalid_input_count >= 2:
            session.mode_type = 'B'
            session.session_status = 'downgraded'
            session.content_log_json = log
            session.save()
            # 降级后返回 B 模式的第一步
            questions_b = (ai_b or {}).get('questions') or []
            total_steps = len(questions_b) or 3
            step_data = _extract_b_mode_step(ai_b, 0, total_steps)
            return Response({'code': 0, 'data': {
                'mode': 'B', 'step_index': 0, 'total_steps': total_steps,
                'next_hint': step_data['hint'], 'options': step_data['options'],
                'downgraded': True, 'downgrade_reason': '连续无效输入，已降级到固定选项引导',
                'is_completed': False,
            }})

    # 获取当前步骤的数据
    if session.mode_type == 'B':
        questions = (ai_b or {}).get('questions') or []
        total_steps = len(questions) or 3
        step_data = _extract_b_mode_step(ai_b, step_index, total_steps)
        # 兜底时 correct_option 为空，跳过正确性判断
        is_correct = None
        if step_data['correct_option']:
            is_correct = user_reply == step_data['correct_option']
        log.setdefault('answers', []).append({
            'step': step_index, 'user_answer': user_reply,
            'correct_answer': step_data['correct_option'], 'is_correct': is_correct,
        })

    if session.mode_type == 'C':
        questions = (ai_c or {}).get('questions') or []
        total_steps = len(questions) or 3
        # 调用 LLM 进行评价（失败时返回兜底文案）
        try:
            evaluation = call_qwen_for_guidance(
                '你是一位耐心的老师，对学生回答给出 1-2 句简明评价与鼓励。',
                f'题目：{q.stem}\n参考答案：{q.answer or "见解析"}\n学生回答：{user_reply}\n请评价。',
            )
        except Exception:
            evaluation = '（AI 评价暂不可用）'
        log.setdefault('answers', []).append({
            'step': step_index, 'user_answer': user_reply,
        })

    # 推进到下一步
    next_step = step_index + 1
    log['step_index'] = next_step
    is_completed = next_step >= total_steps

    # 未完成 → 返回下一步数据
    if not is_completed:
        if session.mode_type == 'B':
            next_data = _extract_b_mode_step(ai_b, next_step, total_steps)
            session.content_log_json = log
            session.save()
            return Response({'code': 0, 'data': {
                'mode': 'B', 'step_index': next_step, 'total_steps': total_steps,
                'is_correct': is_correct,
                'correct_answer': step_data['correct_option'],
                'analysis': step_data['analysis'],
                'next_hint': next_data['hint'],
                'options': next_data['options'],
                'is_completed': False,
            }})
        else:
            next_data = _extract_c_mode_step(ai_c, next_step)
            session.content_log_json = log
            session.save()
            return Response({'code': 0, 'data': {
                'mode': 'C', 'step_index': next_step, 'total_steps': total_steps,
                'evaluation': evaluation,
                'next_hint': next_data['question'],
                'is_completed': False,
            }})

    # 已完成 → 返回总结
    session.session_status = 'completed'
    session.content_log_json = log
    session.save()
    return Response({'code': 0, 'data': {
        'mode': session.mode_type,
        'step_index': next_step, 'total_steps': total_steps,
        'is_completed': True,
        'summary': (ai_b if session.mode_type == 'B' else ai_c).get('summary', ''),
        'final_answer': (ai_b if session.mode_type == 'B' else ai_c).get('final_answer', ''),
        'is_correct': is_correct if session.mode_type == 'B' else None,
        'correct_answer': step_data.get('correct_option', '') if session.mode_type == 'B' else None,
        'analysis': step_data.get('analysis', '') if session.mode_type == 'B' else None,
        'evaluation': evaluation if session.mode_type == 'C' else None,
    }})

    }

})

// 启动引导
async function startGuidance() {
  loading.value = true
  try {
    const res = await studentApi.startGuidance({
      question_id: questionId.value,

    })

    loading.value = false
  }
}

// 选择选项（B 模式）

  submitting.value = true

// 处理回复响应
function handleReplyResponse(data: any) {
  // 处理降级响应
  if (data.downgraded) {
    mode.value = data.mode || 'B'
    messages.value.push({ role: 'system', content: `⚠️ ${data.downgrade_reason || '已降级到固定选项引导模式'}`, type: 'warning' })
    if (data.options) {
      options.value = data.options
    }
    if (data.next_hint) {
      messages.value.push({ role: 'system', content: data.next_hint })
    }
    return
  }

  if (data.mode === 'B') {
    // 显示答案反馈：兜底模式（is_correct=null）不显示正确/错误，只显示提示
    if (data.is_correct === true) {
      messages.value.push({ role: 'system', content: '✅ 回答正确！', type: 'feedback' })
    } else if (data.is_correct === false) {
      messages.value.push({ role: 'system', content: `❌ 回答错误，正确答案是 ${data.correct_answer}`, type: 'feedback' })
    }
    // 兜底模式（is_correct=null）：不显示正确/错误，直接进入下一步
    if (data.analysis) {
      messages.value.push({ role: 'system', content: `解析：${data.analysis}`, type: 'analysis' })
    }
  }

  if (data.evaluation) {
    messages.value.push({ role: 'system', content: `老师评价：${data.evaluation}`, type: 'evaluation' })
  }

  if (data.is_completed) {
    isCompleted.value = true
    summary.value = data.summary || ''
    finalAnswer.value = data.final_answer || ''
    if (summary.value) {
      messages.value.push({ role: 'system', content: `📝 总结：${summary.value}`, type: 'summary' })
    }
    if (finalAnswer.value) {
      messages.value.push({ role: 'system', content: `🎯 最终答案：${finalAnswer.value}`, type: 'final_answer' })
    }
  } else {
    // 更新下一步
    stepIndex.value = data.step_index
    if (data.next_hint) {
      messages.value.push({ role: 'system', content: data.next_hint })
    }
    if (data.options) {
      options.value = data.options
    }
  }
}

// 切换到下一题
async function nextQuestion() {

    questionId.value = questions.value[currentIndex.value].id

    options.value = []
    isCompleted.value = false

    finalAnswer.value = ''
    isCorrect.value = null

    await startGuidance()
    switching.value = false
  } else {
    uni.showToast({ title: '已经是最后一题', icon: 'none' })

}

// 切换到上一题

    currentIndex.value--
    questionId.value = questions.value[currentIndex.value].id
    sessionId.value = 0
    messages.value = []
    options.value = []
    isCompleted.value = false
    summary.value = ''
    finalAnswer.value = ''
    isCorrect.value = null
    selectedOption.value = ''
    await startGuidance()
    switching.value = false
  }
}

### 5.4 加载动画规范

**三种加载状态：**

| 场景 | 动画类型 | 实现方式 | 说明 |

## 7. AI 数据生成与兜底策略

### 7.1 数据生成流程

```
教师上传试卷 → OCR 解析 → ExamQuestion 入库
  │
  ▼
教师在题库页点击"AI 生成"
  ├─ 调用 ai_process_question 接口
  ├─ Celery 任务执行 process_question_full_v2
  │     ├─ probe_and_norm → 规范化题干
  │     ├─ vision_extraction → 读图提取
  │     ├─ solve_mode_b → 生成 B 模式数据 → 存入 ai_answer_b
  │     ├─ solve_mode_c → 生成 C 模式数据 → 存入 ai_answer_c
  │     └─ verify_result → 校验
  └─ ai_answer_b / ai_answer_c 写入数据库
```

## 11. 风险与注意事项

### 11.1 风险点

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| `ai_answer_b.questions` 数量不足 3 个 | 低 | 分步索引越界 | `total_steps = len(questions) or 3`，`_extract_b_mode_step` 中判断索引越界时返回兜底数据 |
| `ai_answer_b.questions[].options` 缺少某个键 | 中 | 选项显示不全 | 只遍历 A/B/C/D 四个键，缺失的键跳过 |
| `ai_answer_c` 数据格式不一致 | 中 | C 模式读取失败 | `_extract_c_mode_step` 中多个 key 兼容读取，失败时返回兜底问题 |
| **兜底时 `correct_option` 为空** | 中 | 正确性判断异常 | `_extract_b_mode_step` 兜底时 `correct_option=''`，`guidance_reply` 中判断 `if step_data['correct_option']` 才做正确性比较，否则 `is_correct=None` |
| 引导页切换题目时旧 session 未清理 | 低 | session 堆积 | 每次启动新 session 前清理 24h 过期 session（已有逻辑） |
| 用户快速点击选项导致重复提交 | 中 | 多次记录 | `submitting` 防抖锁，请求完成后才解锁 |
| **AI 评价接口超时** | 中 | C 模式 evaluation 为空 | `call_qwen_for_guidance` 用 try/except 包裹，失败时返回兜底文案 `'（AI 评价暂不可用）'` |
| **wrongbook 入口无 levelId** | 中 | 无法切题 | 引导页兼容无 `levelId` 的情况：无 levelId 时隐藏"上一题/下一题"按钮，只处理当前题引导 |
| 错误的 AI 解析数据导致前端渲染异常 | 低 | 页面白屏 | 关键区域使用 `v-if` 和兜底文本，不允许 `null` 值渲染 |
| **多选题的 B 模式适配** | 低 | 选项样式不匹配 | 原题是多选题时，B 模式引导的每个步骤仍是单选题（4 选 1），不影响功能，但 UI 上需明确标注"引导题"而非"原题" |

### 11.2 注意事项

1. **数据兼容性**：已有 `ai_answer_b` 数据无需重新生成，因为 AI 数据格式不变，只是读取方式变了
2. **后端自动推荐模式**：`guidance.vue` 启动时传 `mode_type: ''` 让后端根据 `question_type` 自动推荐 B/C 模式。从 `answer.vue` 进入时用户已指定模式，该参数直接传递；从 `wrongbook.vue` 进入时无 mode 参数，留空让后端推荐
3. **错题本入口**：`wrongbook.vue` 导航到引导页时未传 `levelId` 和 `mode`。引导页需兼容无 `levelId` 的情况：隐藏"上一题/下一题"按钮，只处理当前题引导。`mode` 由后端自动推荐
4. **切题时重新初始化**：切换题目时需重置所有引导状态（`sessionId`、`messages`、`options`、`stepIndex`、`isCorrect`、`selectedOption` 等）
5. **引导完成后的提交**：引导完成后，用户可点击"提交答案"按钮直接提交答案（调用 `submitAnswer` 接口），也可点击"返回做题"回到答题页手动提交
6. **AI 数据生成**：建议教师在添加题目后立即触发 AI 生成，确保 `ai_answer_b` 和 `ai_answer_c` 有数据。可通过 `python manage.py generate_ai_guidance --all` 批量生成
7. **兜底模式无正确性判断**：当 `ai_answer_b` 为 null 时使用通用兜底选项，`correct_option` 为空字符串，`guidance_reply` 返回 `is_correct: null`，前端不显示正确/错误标记
8. **C 模式 LLM 调用失败处理**：`call_qwen_for_guidance` 和 `call_qwen_for_guidance_with_question` 均用 try/except 包裹，失败时返回兜底文案，不阻塞流程
9. **多选题的 B 模式引导**：AI 生成的 B 模式每个步骤都是单选题（4 个选项，1 个正确答案），与原题是否为多选题无关。引导页 UI 中需明确标注当前为"引导题"而非"原题"，避免混淆

### 11.3 向后兼容

| 兼容项 | 说明 |
|--------|------|
| 旧版 `ai_answer_b` 数据 | 兼容，读取方式从 `ai_b.get('options')` 改为 `ai_b.get('questions', [])[step_index]` |
| 旧版 `ai_answer_c` 数据 | 兼容，读取 `ai_c.get('questions', [])[step_index]` |
| 旧版 `content_log_json` | 兼容，`log.get('answers', [])` 处理无 `answers` 键的旧数据 |
| 旧版 `guidance.vue` 导航参数 | 兼容，`questionId` + `levelId` + `mode` 参数不变 |
| 旧版 API 返回格式 | 兼容，新增字段不影响旧版解析，旧版字段（`hint`、`options`、`next_hint`）保留 |
