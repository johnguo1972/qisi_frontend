## 三、Docker 部署 PostgreSQL 18.4 + Redis 7.2

> **注意**：Nginx 不包含在此 docker-compose 中，因为它需要 SSL 证书才能启动。
> 证书申请和 Nginx 启动请参见第四、五节。

### 3.1 创建 PostgreSQL 密码密钥文件

```bash

sudo chmod 600 /mnt/datadisk0/qisi/docker-data/postgres_password.txt
```

### 3.2 创建 Docker Compose 配置

创建 `/mnt/datadisk0/qisi/docker-compose.yml`：

```yaml
services:
  postgres:
    image: postgres:18.4-bookworm
    container_name: app-pgsql
    hostname: postgres
    restart: unless-stopped
    ports:
      - "5432:5432"
    volumes:
      - /mnt/datadisk0/qisi/docker-data/pgdata:/var/lib/postgresql
    environment:
      POSTGRES_USER: appuser
      POSTGRES_DB: appdb
      POSTGRES_PASSWORD_FILE: /run/secrets/postgres_password
      TZ: Asia/Shanghai
      PGTZ: Asia/Shanghai
    secrets:
      - postgres_password
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U appuser -d appdb"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7.2-alpine
    container_name: qisi-redis
    hostname: redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - /mnt/datadisk0/qisi/docker-data/redis-data:/data
    command: redis-server --requirepass redis172pwd --appendonly yes
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "redis172pwd", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

secrets:
  postgres_password:
    file: /mnt/datadisk0/qisi/docker-data/postgres_password.txt
```

## 四、申请 HTTPS 证书（Let's Encrypt）

> 在启动 Nginx 之前，必须先获取 SSL 证书。  
> 如果证书已存在（续期场景），请跳过此节，直接进入第五节。

### 4.1 启动临时 HTTP 容器用于 ACME 验证

```bash
docker run -d \
  --name qisi-nginx-init \
  -p 80:80 \
  -v /mnt/datadisk0/docker/volumes/qisi-certbot-www/_data:/var/www/certbot \
  nginx:alpine
```

### 4.2 安装 certbot 并申请证书

```bash
sudo apt install -y certbot
sudo certbot certonly --webroot \
  -w /mnt/datadisk0/docker/volumes/qisi-certbot-www/_data \
  -d qisi.chengxuelu.com \
  --non-interactive --agree-tos -m your-email@example.com
```

> 请将 `your-email@example.com` 替换为你的真实邮箱地址。

### 4.3 将证书复制到 Docker 卷目录

```bash
CERT_SRC=/etc/letsencrypt/live/qisi.chengxuelu.com
CERT_DST=/mnt/datadisk0/docker/volumes/qisi-certbot-certs/_data/live/qisi.chengxuelu.com
sudo mkdir -p $CERT_DST
sudo cp $CERT_SRC/fullchain.pem $CERT_DST/
sudo cp $CERT_SRC/privkey.pem $CERT_DST/
```

### 4.4 停止临时容器

```bash
docker stop qisi-nginx-init && docker rm qisi-nginx-init
```

### 4.5 设置自动续期（crontab）

```bash
sudo crontab -e
```

添加以下行（每月1日凌晨3点检查续期）：

```cron
0 3 1 * * certbot renew --webroot -w /mnt/datadisk0/docker/volumes/qisi-certbot-www/_data --post-hook "cp /etc/letsencrypt/live/qisi.chengxuelu.com/fullchain.pem /mnt/datadisk0/docker/volumes/qisi-certbot-certs/_data/live/qisi.chengxuelu.com/ && cp /etc/letsencrypt/live/qisi.chengxuelu.com/privkey.pem /mnt/datadisk0/docker/volumes/qisi-certbot-certs/_data/live/qisi.chengxuelu.com/ && docker exec qisi-nginx nginx -s reload"
```

## 五、启动 Nginx 反向代理

### 5.1 创建 Nginx 配置

创建 `/opt/qisi/nginx.conf`：

```nginx
events {
    worker_connections 2048;
}

http {
    include /etc/nginx/mime.types;
    resolver 127.0.0.11 valid=10s ipv6=off;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;
    error_log /var/log/nginx/error.log warn;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript
               text/xml application/xml application/xml+rss text/javascript;

    # HTTP → HTTPS 强制跳转
    server {
        listen 80;
        server_name _;

        location /.well-known/acme-challenge/ {
            root /var/www/certbot;
        }
        location / {
            return 301 https://$host$request_uri;
        }
    }

    # HTTPS 服务
    server {
        listen 443 ssl http2;
        server_name _;

        ssl_certificate /etc/nginx/ssl/live/qisi.chengxuelu.com/fullchain.pem;
        ssl_certificate_key /etc/nginx/ssl/live/qisi.chengxuelu.com/privkey.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        ssl_prefer_server_ciphers on;
        ssl_session_cache shared:SSL:10m;
        ssl_session_timeout 10m;

        # OCSP Stapling（提升 HTTPS 握手性能）
        ssl_stapling on;
        ssl_stapling_verify on;
        ssl_trusted_certificate /etc/nginx/ssl/live/qisi.chengxuelu.com/fullchain.pem;

        add_header X-Frame-Options SAMEORIGIN always;
        add_header X-Content-Type-Options nosniff always;
        add_header X-XSS-Protection "1; mode=block" always;
        client_max_body_size 100m;

        # ============================================================
        # 前端 H5 静态页面（SPA）
        # ============================================================
        location / {
            alias /var/www/qisi-frontend-h5/;
            index index.html;
            try_files $uri $uri/ /index.html;

            location ~* \.html$ {
                expires -1;
                add_header Cache-Control "no-cache, must-revalidate";
            }
            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
                expires 30d;
                add_header Cache-Control "public, immutable";
                access_log off;
            }
        }

        # ============================================================
        # 后端 API 代理
        # 重要：proxy_pass 末尾必须带 /api/，否则 Nginx 会剥离 /api/ 前缀
        # 导致 Django 收到 /v1/xxx 而返回 404
        # ============================================================
        location /api/ {
            proxy_pass http://172.18.0.1:8001/api/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_connect_timeout 60s;
            proxy_read_timeout 120s;
            proxy_send_timeout 60s;
            client_max_body_size 100m;
        }

        # ============================================================
        # Django Admin 管理后台
        # 同样：proxy_pass 末尾必须带 /admin/
        # ============================================================
        location /admin/ {
            proxy_pass http://172.18.0.1:8001/admin/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # ============================================================
        # 静态文件
        # ============================================================
        location /static/ {
            alias /var/www/qisi-static/;
            expires 30d;
            add_header Cache-Control "public, immutable";
            access_log off;
        }

        # ============================================================
        # 媒体文件
        # ============================================================
        location /media/ {
            alias /var/www/qisi-media/;
            expires 7d;
            add_header Cache-Control "public, immutable";
            access_log off;
        }

        # ============================================================
        # 健康检查
        # ============================================================
        location /health {
            access_log off;
            return 200 "OK";
            add_header Content-Type text/plain;
        }

        access_log /var/log/nginx/access.log main;
        error_log /var/log/nginx/error.log warn;
    }
}
```

```bash
  --workers 4 \
  --bind 0.0.0.0:8001 \
  --timeout 120 \
  --access-logfile /mnt/datadisk0/qisi/logs/gunicorn-access.log \
  --error-logfile /mnt/datadisk0/qisi/logs/gunicorn-error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

> **注意**：`--bind 0.0.0.0:8001` 监听所有网络接口，包括 Docker 桥接接口，使容器内的 Nginx 可通过 Docker 网关访问宿主机。

### 9.5 排查指南

| 问题 | 排查步骤 |
|------|----------|
| API 返回 502 | 检查 gunicorn 是否运行：`sudo systemctl status qisi-gunicorn`；检查 Nginx 代理地址是否为 `172.18.0.1:8001/api/` |
| API 返回 404 | 检查 Nginx `proxy_pass` 是否带 `/api/` 路径（见 `location /api/` 块） |
| 前端页面 404 | 确认 H5 文件已上传到 `/mnt/datadisk0/qisi/frontend-h5/`；确认 Nginx 使用 `alias` |
| 静态文件 404 | 确认 `collectstatic` 已执行；确认 Nginx 容器挂载了 `/var/www/qisi-static` |
| Nginx 无法启动 | 确认 SSL 证书文件存在：`ls /mnt/datadisk0/docker/volumes/qisi-certbot-certs/_data/live/qisi.chengxuelu.com/` |
| 数据库连接失败 | 确认 `psql -h localhost -p 5432 -U appuser -d appdb` 可连接 |
| Redis 连接失败 | 确认 `redis-cli -h localhost -p 6379 -a redis172pwd ping` 返回 PONG |
