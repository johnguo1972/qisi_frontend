## 一、代码更新后部署流程

每次修改代码后，按以下顺序执行：

### 第 1 步：本地打包后端代码

```bash
# 在本地 Git Bash 执行（项目根目录）
cd D:\yangtze\project\2026\qisi\qisi_frontend
tar --exclude=venv --exclude=node_modules --exclude=.git --exclude=__pycache__ --exclude=*.pyc --exclude=*.pyo --exclude=.coverage --exclude=dump.rdb --exclude=word.zip --exclude=word --exclude=uniapp --exclude=src --exclude=start.bat --exclude=.claude --exclude=*.tar.gz -czf qisi_backend.tar.gz .
```

### 第 2 步：上传到服务器

```bash
scp qisi_backend.tar.gz ubuntu@42.194.195.78:/mnt/datadisk0/qisi/backend/
```

### 第 3 步：服务器更新后端

```bash
# 连接服务器
ssh ubuntu@42.194.195.78

# 解压并更新
cd /mnt/datadisk0/qisi/backend
tar -xzf qisi_backend.tar.gz
rm qisi_backend.tar.gz
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput
sudo systemctl restart qisi-gunicorn
sudo systemctl restart qisi-celery
sudo systemctl restart qisi-celery-beat
```

### 第 4 步：验证

```bash

# 返回 405 表示正常

# 通过 Nginx 验证

# 返回 405 表示正常
```

### 更新前端 H5

## 三、Nginx 管理

### 启动 Nginx 容器

```bash
sudo docker run -d \
  --name qisi-nginx \
  --restart unless-stopped \
  -p 80:80 \
  -p 443:443 \
  -v /opt/qisi/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v /mnt/datadisk0/docker/volumes/qisi-certbot-certs/_data:/etc/nginx/ssl \
  -v /opt/qisi/logs/nginx:/var/log/nginx \
  -v /mnt/datadisk0/docker/volumes/qisi-certbot-www/_data:/var/www/certbot \
  -v /mnt/datadisk0/qisi/frontend-h5:/var/www/qisi-frontend-h5 \
  -v /mnt/datadisk0/qisi/backend/staticfiles:/var/www/qisi-static \
  -v /mnt/datadisk0/qisi/backend/media:/var/www/qisi-media \
  nginx:alpine
```

### 停止

```bash
sudo docker stop qisi-nginx
```

### 重启

```bash
sudo docker restart qisi-nginx
```

### 平滑重载配置（修改 nginx.conf 后执行，零中断）

```bash
sudo docker exec qisi-nginx nginx -s reload
```

### 查看状态

```bash
sudo docker ps | grep qisi-nginx
```

### 检查配置语法

```bash
sudo docker exec qisi-nginx nginx -t
```

## 五、PostgreSQL 操作

### 连接数据库

```bash
# 通过容器连接
sudo docker exec -it app-pgsql psql -U appuser -d appdb

# 通过宿主机连接（需要安装 psql）
psql -h localhost -p 5432 -U appuser -d appdb
```

### 常用 SQL

```sql
-- 查看所有表
\dt

-- 查看表结构
\d 表名

-- 查看所有数据库
\l

-- 查看所有用户
\du

-- 退出
\q
```

### 修改密码

```bash
sudo docker exec app-pgsql psql -U appuser -d appdb -c "ALTER USER appuser PASSWORD 'pgsql!1804pwd';"
```

### 备份数据库

```bash
sudo docker exec app-pgsql pg_dump -U appuser appdb > /mnt/datadisk0/qisi/backups/backup_$(date +%Y%m%d).sql
```

### 恢复数据库

```bash
cat /mnt/datadisk0/qisi/backups/backup_20260722.sql | sudo docker exec -i app-pgsql psql -U appuser -d appdb
```

## 六、Redis 操作

### 连接 Redis

```bash
# 通过容器连接
sudo docker exec -it app-redis redis-cli -a redis172pwd

# 通过宿主机连接
redis-cli -h localhost -p 6379 -a redis172pwd
```

### 常用命令

```bash
# 测试连接
sudo docker exec app-redis redis-cli -a redis172pwd ping

# 查看所有 key
sudo docker exec app-redis redis-cli -a redis172pwd KEYS "*"

# 查看 key 的值
sudo docker exec app-redis redis-cli -a redis172pwd GET 键名

# 查看 Redis 版本
sudo docker exec app-redis redis-cli -a redis172pwd INFO server | grep redis_version
```

## 十、快速排查命令

```bash
# 1. 检查所有服务是否运行
sudo docker ps
sudo systemctl status qisi-gunicorn --no-pager | grep Active

# 3. 检查 Nginx 代理
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" https://qisi.chengxuelu.com/api/v1/auth/login

# 4. 检查数据库连接
sudo docker exec app-pgsql pg_isready -U appuser -d appdb

# 5. 检查 Redis 连接
sudo docker exec app-redis redis-cli -a redis172pwd ping

# 6. 查看最新错误日志
tail -20 /mnt/datadisk0/qisi/logs/gunicorn-error.log
```
