## 二、启动本地 Docker 服务

### 2.1 停止旧容器（如果存在）

```bash
# 如果之前有旧版的 qisi-postgres（PG 16），先停止并删除
docker stop qisi-postgres qisi-redis 2>/dev/null
docker rm qisi-postgres qisi-redis 2>/dev/null

# 删除旧数据卷（可选，如果不需要保留旧数据）
docker volume rm qisi_postgres_pgdata 2>/dev/null
```

### 2.2 启动新容器

```bash
# 在项目根目录执行
docker compose -f word\260722_项目新部署指南\local\docker-compose.local.yml up -d

# 检查容器状态
docker ps

# 确认 PostgreSQL 连接正常
docker exec qisi-postgres psql -U appuser -d appdb -c "SELECT version();"

# 确认 Redis 连接正常
docker exec qisi-redis redis-cli -a redis172pwd ping
```

## 七、常见问题

### Q: 旧数据如何迁移到新数据库？

如果旧数据库（`qisi_ai_tutor`）中有需要保留的数据：

```bash
# 1. 从旧容器导出
docker exec qisi-postgres-old pg_dump -U qisi_admin qisi_ai_tutor > backup.sql

# 2. 导入到新容器
docker exec -i qisi-postgres psql -U appuser -d appdb < backup.sql

# 3. 运行迁移（更新 django_migrations 表）
python manage.py migrate --fake-initial
```

### Q: 端口冲突怎么办？

如果 5432 或 6379 端口已被占用：

```bash
# 修改 docker-compose.local.yml 中的端口映射
# 例如: "5433:5432" 将宿主机 5433 映射到容器 5432
# 同时修改 .env 中的 DB_PORT=5433
```

### Q: 如何查看数据库内容？

```bash
# 直接连接容器内的 PostgreSQL
docker exec -it qisi-postgres psql -U appuser -d appdb

# 查看所有表
\dt

# 退出
\q
```
