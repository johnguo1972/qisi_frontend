## 二、服务器现状分析

### 2.1 已运行的服务

| 容器名 | 镜像 | 端口 | 用途 |
|--------|------|------|------|
| `qisi-nginx` | nginx:alpine | 80 / 443 | 共享 Nginx 反向代理 |
| `qisi-postgres` | postgres | 5432 | PostgreSQL 数据库 |
| `qisi-redis` | redis | 6379 | Redis 缓存/队列 |
| `tiku-web` | a56ee61cd6ee | 8000 → 8000 | 已有 Django 题库项目 |
| `tiku-celery-worker` | a56ee61cd6ee | - | 已有 Celery Worker |
| `tiku-celery-beat` | a56ee61cd6ee | - | 已有 Celery Beat |
| `qisi-backapi` | ddd31ed96783 | 3000 | Next.js 后端项目 |
| `qisi-certbot` | certbot | - | Let's Encrypt 证书续期 |

### 2.2 已有 Nginx 配置要点

- **Nginx 运行在 Docker 容器中**，配置挂载在宿主机路径：`/opt/qisi/nginx.conf`
- 已有 HTTPS 证书（Let's Encrypt），证书路径：`/etc/nginx/ssl/live/qisi.chengxuelu.com/`
- 已有路由规则：
  - `/api/review/`  → tiku-web:8000
  - `/api/papers/`  → tiku-web:8000
  - `/api/`         → qisi-backapi:3000
  - `/`             → tiku-web:8000（默认）
- 所有服务在 Docker 网络 `qisi-network`（172.18.0.0/16）内互联

### 2.3 本项目部署策略

基于以上现状，本项目的部署策略为：

| 策略 | 原因 |
|------|------|
| **后端直接运行在宿主机**（非 Docker） | 避免与现有 Docker 容器冲突，方便独立管理 |
| **后端监听 8001 端口** | 8000 已被 tiku 占用 |
| **修改 `/opt/qisi/nginx.conf`** | 在现有 Nginx 中添加 `/study/` 路由，不创建新 Nginx |
| **数据库使用现有 qisi-postgres** | 通过 `172.18.0.1` 网关地址连接 |
| **Redis 使用现有 qisi-redis** | 通过 `172.18.0.1` 网关地址连接 |

### 4.8 配置 systemd 服务（开机自启 + 进程守护）

#### 4.8.1 创建 Gunicorn 服务

```bash
sudo tee /etc/systemd/system/qisi-gunicorn.service > /dev/null << 'EOF'
[Unit]
Description=Qisi Django Gunicorn Service
After=network.target docker.service
Wants=docker.service

[Service]
Type=simple
User=ubuntu
Group=ubuntu
WorkingDirectory=/mnt/diskdata0/qisi/backend
Environment="PATH=/mnt/diskdata0/qisi/backend/venv/bin"
ExecStart=/mnt/diskdata0/qisi/backend/venv/bin/gunicorn config.wsgi:application \
  --workers 4 \
  --bind 127.0.0.1:8001 \
  --timeout 120 \
  --access-logfile /mnt/diskdata0/qisi/logs/gunicorn-access.log \
  --error-logfile /mnt/diskdata0/qisi/logs/gunicorn-error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

### 5.3 添加本项目路由规则

在现有 `nginx.conf` 的 HTTPS server 块中（`server { listen 443 ssl; ... }` 内部），在 `client_max_body_size 50M;` 之后、`# === tiku Django API endpoints ===` 之前，添加以下内容：

```nginx
    # ============================================================
    # 齐思 · A自习室 (qisi_frontend) — 部署在 /study 子路径
    # 后端监听宿主机 127.0.0.1:8001
    # ============================================================

    # 前端 H5 静态页面（SPA 单页应用）
    location /study/ {
        root /mnt/diskdata0/qisi/frontend-h5;
        index index.html;
        try_files $uri $uri/ /study/index.html;

        # HTML 不做缓存（确保每次更新后用户能拿到最新版本）
        location ~* \.html$ {
            expires -1;
            add_header Cache-Control "no-cache, must-revalidate";
        }

        # JS/CSS/图片等静态资源强缓存（文件名含 hash）
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
            access_log off;
        }
    }

    # 后端 API 代理（Django REST API）
    location /study/api/ {
        proxy_pass http://172.18.0.1:8001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 120s;
        proxy_send_timeout 60s;
        client_max_body_size 100m;
    }

    # Django Admin 管理后台
    location /study/admin/ {
        proxy_pass http://172.18.0.1:8001/admin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Django 静态文件（collectstatic 产物）
    location /study/static/ {
        alias /mnt/diskdata0/qisi/backend/staticfiles/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # 媒体文件（用户上传的试卷、图片等）
    location /study/media/ {
        alias /mnt/diskdata0/qisi/backend/media/;
        expires 7d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }
```

或使用 MobaXterm 直接拖拽上传。

### 6.3 打包 APK（华为平板端）

#### 6.3.1 使用 HBuilderX 云打包（推荐）

**步骤：**

1. **下载安装 HBuilderX**
   - 访问 https://www.dcloud.io/hbuilderx.html 下载安装
   - 安装后打开 HBuilderX，登录 DCloud 账号

2. **导入项目**
   - 菜单：文件 → 导入 → 导入本地项目
   - 选择 `D:\yangtze\project\2026\qisi\qisi_frontend\uniapp` 目录

3. **配置 manifest.json（App 打包配置）**

   编辑 `uniapp/src/manifest.json`，配置 App 相关参数：

   ```json
   {
     "name": "A自习室",
     "appid": "",  // 留空，HBuilderX 会自动生成
     "description": "A自习室题库系统",
     "versionName": "1.0.0",
     "versionCode": "100",
     "transformPx": false,
     "app-plus": {
       "modules": {},
       "distribute": {
         "android": {
           "permissions": [
             "<uses-permission android:name=\"android.permission.INTERNET\"/>",
             "<uses-permission android:name=\"android.permission.ACCESS_NETWORK_STATE\"/>",
             "<uses-permission android:name=\"android.permission.READ_EXTERNAL_STORAGE\"/>",
             "<uses-permission android:name=\"android.permission.WRITE_EXTERNAL_STORAGE\"/>"
           ]
         }
       }
     },
     "networkTimeout": {
       "request": 30000,
       "downloadFile": 60000,
       "uploadFile": 60000
     },
     "h5": {
       "router": {
         "mode": "hash",
         "base": "/study/"
       }
     }
   }
   ```

4. **配置 API 请求地址（关键）**

   APK 中发起的请求直接访问服务器，不走相对路径。需要修改 `uniapp/src/utils/request.ts` 中的 `BASE_URL`，使用 **条件编译** 区分环境：

   ```typescript
   // #ifdef APP-PLUS
   // App 环境（APK）：使用完整域名
   const BASE_URL = 'https://qisi.chengxuelu.com/study/api/v1'
   // #endif
   // #ifndef APP-PLUS
   // H5 环境（浏览器）：使用相对路径，由 Nginx 代理
   const BASE_URL = '/api/v1'
   // #endif
   ```

   > **说明**：`#ifdef APP-PLUS` 是 uni-app 的条件编译语法，仅在 App 打包时生效，H5 环境不受影响。这样开发时依然使用相对路径，APK 中使用完整域名。

5. **原生 App 云打包**
   - 在 HBuilderX 中，菜单：**发行 → 原生App-云打包**
   - 选择平台：**Android**
   - 打包类型：**公共测试证书**（或使用自有证书）
   - 点击 **"打包"**
   - 等待打包完成（约 5-15 分钟）
   - 打包完成后会自动下载 APK 文件

6. **安装到华为平板**
   - 将 APK 文件通过 USB 数据线或微信/QQ 传输到华为平板
   - 在平板上打开文件管理器，找到 APK 文件
   - 点击安装（如提示"未知来源应用"，需要在设置中允许）
   - 安装完成后打开应用，测试登录和各页面功能

### 8.6 排查指南

| 问题 | 排查步骤 |
|------|----------|
| 后端无法启动 | 检查 `journalctl -u qisi-gunicorn -f` 查看错误日志 |
| API 返回 502 | 检查 Nginx 代理地址是否正确，后端是否运行 |
| 数据库连接失败 | 检查 `.env` 中 `DB_HOST=172.18.0.1`，确认 Docker 网关地址 |
| Redis 连接失败 | 检查 `.env` 中 Redis 地址和密码，确认 Redis 容器运行中 |
| 前端页面空白 | 检查浏览器控制台，确认 `manifest.json` 中 `base` 是否为 `/study/` |
| APK 无法连接 | 确认 APK 中 `BASE_URL` 使用条件编译设置为完整域名 |
| Nginx 配置错误 | 检查 `sudo docker logs qisi-nginx` 查看容器错误日志 |

### 8.7 不影响其他项目的关键原则

| 原则 | 说明 |
|------|------|
| **不修改已有配置** | 只向 `/opt/qisi/nginx.conf` 添加新 location 块，不修改已有规则 |
| **使用独立端口** | 后端监听 8001，不占用 8000 |
| **使用独立 Redis DB** | 使用 1/2/3 号数据库，不占用 0 号 |
| **共享数据库需谨慎** | 表名不会冲突（Django 自动创建），但迁移前建议备份 |
