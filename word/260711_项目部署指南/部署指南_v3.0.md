## 三、修复清单（v2.0 → v3.0）

| # | 问题 | 严重程度 | 修复方式 |
|---|------|---------|---------|
| 1 | Gunicorn 绑定 `127.0.0.1:8001`，Docker Nginx 无法访问 | 🔴 严重 | 改为 `0.0.0.0:8001` |
| 2 | Nginx 容器无法访问宿主机文件路径 | 🔴 严重 | 新增三个 bind mount 到容器 |
| 3 | Nginx `root` 指令导致路径拼接错误 | 🔴 严重 | 改为 `alias` |
| 4 | Django STATIC_URL/MEDIA_URL 缺少 `/study/` 前缀 | 🔴 严重 | 添加 `FORCE_SCRIPT_NAME = '/study'` |
| 5 | 路径 `/mnt/diskdata0/` 与现有 `/mnt/datadisk0/` 不一致 | 🟠 高 | 统一使用 `/mnt/datadisk0/` |
| 6 | 通过 `172.18.0.1` 网关连接数据库/Redis 不可靠 | 🟠 高 | 改为 `localhost`（端口已发布到宿主机） |
| 7 | `docker restart qisi-nginx` 导致全站中断 | 🟠 高 | 改为 `nginx -s reload` |
| 8 | Django Admin 后台 URL 缺少 `/study/` 前缀 | 🟡 中 | 由 `FORCE_SCRIPT_NAME` 统一修复 |
| 9 | 缺少 Nginx 配置语法检查步骤 | 🟢 低 | 增加 `nginx -t` 检查步骤 |

### 5.6 配置 systemd 服务

#### 5.6.1 Gunicorn 服务（监听 0.0.0.0:8001）

```bash
sudo tee /etc/systemd/system/qisi-gunicorn.service > /dev/null << 'EOF'
[Unit]
Description=Qisi Django Gunicorn Service
After=network.target docker.service
Wants=docker.service

[Service]
Type=simple
User=ubuntu
Group=ubuntu
WorkingDirectory=/mnt/datadisk0/qisi/backend
Environment="PATH=/mnt/datadisk0/qisi/backend/venv/bin"
ExecStartPre=/bin/sleep 5
ExecStart=/mnt/datadisk0/qisi/backend/venv/bin/gunicorn config.wsgi:application \
  --workers 4 \
  --bind 0.0.0.0:8001 \
  --timeout 120 \
  --access-logfile /mnt/datadisk0/qisi/logs/gunicorn-access.log \
  --error-logfile /mnt/datadisk0/qisi/logs/gunicorn-error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

> **关键修复**：`--bind 0.0.0.0:8001` 而非 `127.0.0.1:8001`。因为 Nginx 运行在 Docker 容器中，需要通过 Docker 桥接网络访问宿主机。`0.0.0.0` 监听所有网络接口，包括 Docker 桥接接口。

## 六、修改 Django 配置

> **说明**：本项目部署在 `/study` 子路径下，Django 需要知道这个前缀才能正确生成 URL。以下修改是标准部署配置，不涉及业务逻辑代码变更。

### 6.1 修改 settings.py

编辑 `config/settings.py`，在文件末尾（`ALIYUN_OSS_ENDPOINT` 之后）添加以下内容：

```bash
cd /mnt/datadisk0/qisi/backend
nano config/settings.py
```

在文件末尾添加：

```python
# ========== 部署配置（/study 子路径前缀） ==========
# 部署在 qisi.chengxuelu.com/study 子路径下
# FORCE_SCRIPT_NAME 让 Django 在所有自生成的 URL 前添加 /study 前缀
FORCE_SCRIPT_NAME = '/study'

# 静态文件 URL（需包含 /study 前缀，与 Nginx 路由匹配）
STATIC_URL = '/study/static/'

# 媒体文件 URL（需包含 /study 前缀，与 Nginx 路由匹配）
MEDIA_URL = '/study/media/'
```

> **为什么需要这些变更？**  
> 部署文档 v2.0 遗漏了此配置，导致 Django 管理后台、静态文件、媒体文件的 URL 均缺少 `/study/` 前缀，浏览器会请求到错误路径，被已有项目（tiku）的 `/` 路由拦截，导致页面样式错乱、文件无法加载。

### 6.2 重启 Gunicorn 使配置生效

```bash
sudo systemctl restart qisi-gunicorn
```

## 七、修改 Nginx 容器配置

### 7.1 备份现有配置

```bash
sudo cp /opt/qisi/nginx.conf /opt/qisi/nginx.conf.bak.$(date +%Y%m%d)
```

### 7.2 添加本项目路由到 nginx.conf

编辑 `/opt/qisi/nginx.conf`：

```bash
sudo nano /opt/qisi/nginx.conf
```

在 HTTPS server 块中（`server { listen 443 ssl; ... }` 内部），在 `client_max_body_size 50M;` 之后、`# === tiku Django API endpoints ===` 之前，添加以下内容：

```nginx
    # ============================================================
    # 齐思 · A自习室 (qisi_frontend) — 部署在 /study 子路径
    # 后端监听 0.0.0.0:8001（通过 Docker 网关访问宿主机）
    # ============================================================

    # 前端 H5 静态页面（SPA 单页应用）
    # 注意：使用 alias 而非 root，避免路径拼接错误
    location /study/ {
        alias /var/www/qisi-frontend-h5/;
        index index.html;
        try_files $uri $uri/ /study/index.html;

        # HTML 不做缓存
        location ~* \.html$ {
            expires -1;
            add_header Cache-Control "no-cache, must-revalidate";
        }

        # JS/CSS/图片等静态资源强缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
            access_log off;
        }
    }

    # 后端 API 代理
    location /study/api/ {
        proxy_pass http://172.18.0.1:8001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 120s;
        proxy_send_timeout 60s;
        client_max_body_size 100m;
    }

    # Django Admin 管理后台
    location /study/admin/ {
        proxy_pass http://172.18.0.1:8001/admin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Django 静态文件
    location /study/static/ {
        alias /var/www/qisi-static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # 媒体文件（用户上传）
    location /study/media/ {
        alias /var/www/qisi-media/;
        expires 7d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }
```

> **关键修复说明**：  
> 1. `/study/` 使用 `alias` 而非 `root`，避免 Nginx 将 `/study/` 拼接到文件路径中  
> 2. 容器内路径 (`/var/www/qisi-*`) 与宿主机路径 (`/mnt/datadisk0/qisi/...`) 不同，通过 bind mount 映射  
> 3. `proxy_pass` 指向 `172.18.0.1:8001`（Docker 网关地址），gunicorn 绑定 `0.0.0.0:8001` 保证可达

### 7.3 检查 Nginx 配置语法

```bash
# 在容器内检查配置语法（不中断现有服务）
sudo docker exec qisi-nginx nginx -t
```

如果输出 `syntax is ok` 和 `test is successful`，则配置正确。

### 7.4 添加新目录挂载到 Nginx 容器

> **为什么需要这一步？**  
> 现有 Nginx 容器只挂载了 tiku 项目的目录。我们的项目目录 `/mnt/datadisk0/qisi/...` 不在容器内，Nginx 无法访问。需要将新目录挂载到容器中。

**查看当前容器启动参数：**

```bash
# 获取当前容器的详细配置
sudo docker inspect qisi-nginx --format='{{json .HostConfig.Binds}}' | python3 -m json.tool
```

**重建 Nginx 容器（添加新挂载）：**

```bash
# 停止并删除现有容器
sudo docker stop qisi-nginx
sudo docker rm qisi-nginx

# 重新创建容器，添加 qisi 项目的挂载
sudo docker run -d \
  --name qisi-nginx \
  --restart unless-stopped \
  --network qisi-network \
  -p 80:80 \
  -p 443:443 \
  -v /opt/qisi/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v /mnt/datadisk0/docker/volumes/qisi-certbot-certs/_data:/etc/nginx/ssl \
  -v /opt/qisi/logs/nginx:/var/log/nginx \
  -v /mnt/datadisk0/docker/volumes/qisi-certbot-www/_data:/var/www/certbot \
  -v /mnt/datadisk0/tiku-media:/var/www/tiku-media \
  -v /mnt/datadisk0/tiku-static:/var/www/tiku-static \
  -v /mnt/datadisk0/qisi/frontend-h5:/var/www/qisi-frontend-h5 \
  -v /mnt/datadisk0/qisi/backend/staticfiles:/var/www/qisi-static \
  -v /mnt/datadisk0/qisi/backend/media:/var/www/qisi-media \
  nginx:alpine
```

> **注意**：新增的挂载是：
> - `/mnt/datadisk0/qisi/frontend-h5` → `/var/www/qisi-frontend-h5`
> - `/mnt/datadisk0/qisi/backend/staticfiles` → `/var/www/qisi-static`
> - `/mnt/datadisk0/qisi/backend/media` → `/var/www/qisi-media`

### 7.5 平滑重载 Nginx 配置

```bash
# 使用 reload 而非 restart，避免中断现有服务
sudo docker exec qisi-nginx nginx -s reload
```

> **为什么用 `reload` 而非 `restart`？**  
> `docker restart` 会完全停止容器再启动，导致 **所有项目**（tiku、backapi）短暂中断。  
> `nginx -s reload` 在不停止进程的情况下重新加载配置，**零中断**。

### 7.6 验证 Nginx 配置生效

```bash
# 查看容器日志确认启动正常
sudo docker logs qisi-nginx --tail 20
```

### 10.6 不影响其他项目的关键原则

|------|------|

| 平滑重载 Nginx | 使用 `nginx -s reload` 而非 `docker restart`，零中断 |
