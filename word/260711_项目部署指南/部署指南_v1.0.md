## 一、部署架构总览

```text
┌──────────────────────────────────────────────────┐
│                    用户                           │
│  华为平板 (APK) / 浏览器 → qisi.chengxuelu.com   │
└──────────────┬───────────────────────────────────┘
               │
               ▼

│              Nginx (反向代理)                      │
│  端口 443 (HTTPS) / 80 (HTTP → 重定向 HTTPS)     │
│  qisi.chengxuelu.com/study → 后端 127.0.0.1:8000 │
└──────────────┬───────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────┐
│           Gunicorn (Django WSGI)                  │
│  127.0.0.1:8000                                  │
│  workers=4                                       │
├──────────────────────────────────────────────────┤
```

| PostgreSQL | 5432 | 数据库，仅监听 127.0.0.1 |

## 四、配置 Nginx 反向代理

### 4.1 创建 Nginx 站点配置

由于域名使用 `qisi.chengxuelu.com/study` 子路径，需要确保 `/study` 路径下的所有请求都代理到后端，并且 `/study` 自身也需要返回前端页面。

```bash
sudo nano /etc/nginx/sites-available/qisi
```

写入以下配置：

```nginx
# 齐思 · A自习室 - 域名 qisi.chengxuelu.com/study
server {
    listen 80;
    server_name qisi.chengxuelu.com;

    # ========== 后端 API 代理 ==========
    # API 请求代理到 Django
    location /study/api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 120s;
    }

    # Admin 后台
    location /study/admin/ {
        proxy_pass http://127.0.0.1:8000/admin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # ========== 静态文件 ==========
    # Django 静态文件
    location /study/static/ {
        alias /mnt/diskdata0/qisi/backend/staticfiles/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 媒体文件（用户上传）
    location /study/media/ {
        alias /mnt/diskdata0/qisi/backend/media/;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # ========== 前端页面 ==========
    # 前端 H5 构建产物（托管在服务器上）
    location /study/ {
        # 如果前端静态文件存在则直接返回，否则 fallback 到 index.html（SPA 路由）
        root /mnt/diskdata0/qisi/frontend-h5;
        index index.html;
        try_files $uri $uri/ /study/index.html;

        # HTML 不做缓存
        location ~* \.html$ {
            expires -1;
            add_header Cache-Control "no-cache, must-revalidate";
        }
        
        # 静态资源缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }
}
```

## 五、前端 H5 构建与 APK 打包

### 5.1 前端架构说明

前端使用 uni-app（Vue 3）框架，支持多端编译。当前项目通过 `H5` 模式在浏览器中运行，并通过 **HBuilderX** 打包为 Android APK 安装到华为平板。

### 5.2 构建 H5 版本（部署到服务器）

#### 5.2.1 修改前端 API 配置

在 `uniapp/src/utils/request.ts` 中，`BASE_URL` 已经是 `/api/v1` 的相对路径，前端会自动使用当前域名。但部署到 `/study` 子路径下时，需要确保 API 请求正确。

**关键点**：由于 Nginx 配置中 `/study/api/` 被代理到 `/api/`，前端请求 `/api/v1/...` 会被正确转发。但前端部署在 `/study/` 下时，需要确保前端资源路径正确。

#### 5.2.2 修改 manifest.json 配置 H5 路由

编辑 `uniapp/src/manifest.json`，设置 H5 路由 base：

```json
{
  "h5": {
    "router": {
      "mode": "hash",
      "base": "/study/"
    },
    "devServer": {
      "port": 5273
    }
  }
}
```

### 5.3 打包 APK（华为平板端）

#### 5.3.1 方式一：使用 HBuilderX 打包（推荐）

HBuilderX 是 DCloud 官方提供的 uni-app 开发工具，支持一键打包为 APK。

**步骤：**

1. **下载安装 HBuilderX**
   - 访问 https://www.dcloud.io/hbuilderx.html 下载安装
   - 安装后打开 HBuilderX

2. **导入项目**
   - 菜单：文件 → 导入 → 导入本地项目
   - 选择 `D:\yangtze\project\2026\qisi\qisi_frontend\uniapp` 目录

3. **配置 manifest.json（App 打包配置）**
   
   在 `uniapp/src/manifest.json` 中，配置 App 相关参数：
   
   ```json
   {
   ```

4. **配置 API 请求地址（重要）**

   APK 中发起的请求直接访问服务器，不走相对路径。需要修改 `uniapp/src/utils/request.ts` 中的 `BASE_URL`，使其支持生产环境地址。

   推荐修改方式：在 `request.ts` 中根据环境变量或运行时判断设置 `BASE_URL`：

   ```typescript
   // 判断是否在 App 环境（APK 中运行）
   // #ifdef APP-PLUS
   const BASE_URL = 'https://qisi.chengxuelu.com/study/api/v1'
   // #endif
   // #ifndef APP-PLUS
   const BASE_URL = '/api/v1'
   // #endif
   ```

   > **说明**：`#ifdef APP-PLUS` 是 uni-app 的条件编译语法，仅在 App 打包时生效，H5 环境不受影响。

5. **原生 App 云打包**
   
   **方式一：在 HBuilderX 中操作**
   - 菜单：发行 → 原生App-云打包
   - 选择 Android 平台
   - 打包类型：选择"公共测试证书"（或使用自有证书）
   - 点击"打包"
   - 等待打包完成（约 5-15 分钟）
   - 下载 APK 文件

   **方式二：使用命令行打包**
   ```bash
   cd uniapp
   npm run build:app
   ```
   然后在 HBuilderX 中打开项目，选择"发行 → 原生App-云打包"。

6. **安装到华为平板**
   - 将 APK 文件传输到华为平板
   - 在平板上点击 APK 文件安装
   - 安装完成后打开应用，确认能正常打开登录页面

#### 5.3.2 方式二：使用本地打包（离线打包）

如果需要离线打包，需要下载 Android Studio 和 uni-app 离线 SDK：

1. 下载 uni-app 离线 SDK（NativeSDK）

4. 生成 APK

> 离线打包较为复杂，推荐使用云打包方式。

### 7.2 代码更新部署

```bash

# 收集静态文件

# 重启服务

sudo systemctl restart qisi-celery
```

```bash

pg_dump -U qisi_admin -h localhost qisi_ai_tutor > /mnt/diskdata0/qisi/backups/qisi_ai_tutor_$(date +%Y%m%d).sql

# 添加以下行（每天凌晨3点备份）

```

### 7.4 日志轮转

创建 `/etc/logrotate.d/qisi`：

```bash
sudo nano /etc/logrotate.d/qisi
```

```
/mnt/diskdata0/qisi/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
}
```

### 7.5 端口冲突排查

如果服务器上已有其他服务占用了所需端口：

```bash
# 查看端口占用

# 如需修改 Django 端口，修改 systemd 服务文件中的 --bind 参数
# 然后重启服务
sudo systemctl daemon-reload
sudo systemctl restart qisi-gunicorn
```

```bash

# 注意：确保 8000 端口不对外开放（仅监听 127.0.0.1）
```
