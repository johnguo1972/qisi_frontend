# 验证修复 - 阶段2前置问题修复清单

## 已修复的问题

### 1. ✅ 错题本跳转参数错误（前端bug）
**问题**：点击错题本报"缺少题目id"
**原因**：wrongbook.vue传递参数`wrongId`，但guidance.vue期望`questionId`
**修复**：
- 文件：`uniapp/src/pages/student/wrongbook.vue`
- 修改第31行：`@click="goDetail(item)`（传递完整item对象）
- 修改第77-79行：goDetail函数从item中获取question_id

### 2. ✅ 单选题可多选（前端bug）
**问题**：同类题单选题可以多选
**原因**：toggleOption函数未区分单选/多选题型
**修复**：
- 文件：`uniapp/src/pages/student/variant-practice.vue`
- 修改第78-87行：toggleOption函数增加单选题判断逻辑
  - 单选题：清空后只选当前选项
  - 多选题：追加选项

### 3. ✅ 多选题无选项数据（后端数据过滤）
**问题**：题目10（多选题）返回options为空数组
**原因**：题目导入时选项数据缺失
**修复**：
- 文件：`apps/wrongbook/services.py`
- 修改find_variant_questions函数：过滤掉没有选项的选择题
- 防止数据不完整的题目推荐给学生

### 4. ✅ 首页任务筛选逻辑（后端bug）
**问题**：首页任务为空（实际有任务进度）
**原因**：class_id=0被当作有效筛选条件，过滤掉了所有任务
**修复**：
- 文件：`apps/study/student_views.py`
- 修改第35-37行：`if class_id and int(class_id) > 0`才进行筛选

## 问题2说明：错题本有数据但任务进度未推进

**性质**：数据问题（非bug）

**原因分析**：
- 学生通过**非任务途径答题**（自由答题，level_id=null）
- AnswerAttempt记录的mission_id为null
- 错题本有数据，但任务进度不推进（符合业务逻辑）

**正确使用方式**：
学生应通过"首页 → 任务 → 关卡 → 答题"路径答题，才能推进任务进度。
