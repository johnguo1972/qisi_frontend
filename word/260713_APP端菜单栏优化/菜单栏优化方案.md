## 一、问题分析

### 1.1 根因

在 uni-app 中，`uni.navigateTo()` 在不同环境下的行为不同：

| 环境 | 底层机制 | 侧边栏行为 |
|------|---------|-----------|
| H5 浏览器 | 单页应用（SPA），hash 路由 | 组件常驻，只更新 `<router-view>`，侧边栏固定不动 |
| APP（APK） | 原生页面栈，`uni.navigateTo` 推入一个新原生页面 | 侧边栏跟随页面重新渲染，出现闪烁 |

### 1.2 当前代码中的问题

两个侧边栏组件均使用 `uni.navigateTo` 进行菜单切换：

**`StudentSidebar.vue`（第 114-118 行）：**
```javascript
function goHome() { uni.navigateTo({ url: '/pages/student/home' }) }
function goWrongBook() { uni.navigateTo({ url: '/pages/student/wrongbook' }) }
function goKnowledgeGraph() { uni.navigateTo({ url: '/pages/student/knowledge-graph' }) }
function goGrowth() { uni.navigateTo({ url: '/pages/student/growth' }) }
function goJoinClass() { uni.navigateTo({ url: '/pages/student/join-class' }) }
```

**`TeacherSidebar.vue`（第 67-73 行）：**
```javascript
function goWorkbench() { uni.navigateTo({ url: '/pages/teacher/workbench' }) }
function goImport() { uni.navigateTo({ url: '/pages/teacher/import' }) }
function goNewQuestion() { uni.navigateTo({ url: '/pages/teacher/new-question' }) }
function goBank() { uni.navigateTo({ url: '/pages/teacher/bank' }) }
function goFavorites() { uni.navigateTo({ url: '/pages/teacher/favorites' }) }
function goClasses() { uni.navigateTo({ url: '/pages/teacher/my-classes' }) }
function goMissionList() { uni.navigateTo({ url: '/pages/teacher/workbench' }) }
```

## 二、核心思路

### 2.1 架构变更

将侧边栏菜单页面从"独立页面 + uni.navigateTo 跳转"改为 **"Layout 页面 + Vue 动态组件切换"**：

```text
修改前：
  /pages/student/home        ← 独立页面，含侧边栏
  /pages/student/wrongbook   ← 独立页面，含侧边栏
  ↓ 点击菜单 → uni.navigateTo 跳转 → 整个页面刷新

修改后：
  /pages/student/layout      ← 唯一页面，包含侧边栏 + 动态内容区
    ├── <StudentSidebar />    ← 侧边栏固定，不刷新
    ├── <component :is="home" />       ← 动态切换
    ├── <component :is="wrongbook" />  ← 动态切换
    └── ...
  ↓ 点击菜单 → emit 事件 → component 切换 → 只换内容区
```

### 2.2 关键原则

| 原则 | 说明 |
|------|------|
| **不修改子页面** | 12 个教师端子页面保持原样，包括侧边栏和 pages.json 注册 |
| **不修改全屏页面** | 知识图谱、关卡等全屏页面保持 navigateTo 不变 |
| **不修改路由** | 子页面之间的 navigateTo 路径不变，依然可正常跳转 |
| **只影响侧边栏菜单** | 仅 11 个侧边栏菜单页面改为组件切换 |

## 四、详细实施步骤

### 步骤 1：新建 Layout 页面

#### 1.1 新建 `pages/student/layout.vue`

**文件路径：** `uniapp/src/pages/student/layout.vue`

```vue
<template>
  <view class="layout">
    <StudentSidebar :activeItem="currentPage" @navigate="switchPage" />
    <view class="content-area">
      <component :is="currentComponent" />
    </view>
  </view>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue'
import StudentSidebar from '@/components/StudentSidebar.vue'
import HomePage from './home.vue'
import WrongbookPage from './wrongbook.vue'
import GrowthPage from './growth.vue'
import JoinClassPage from './join-class.vue'

const currentPage = ref('home')

const components: Record<string, any> = {
  home: HomePage,
  wrongbook: WrongbookPage,
  growth: GrowthPage,
  'join-class': JoinClassPage,
}

const currentComponent = computed(() => components[currentPage.value])

function switchPage(page: string) {
  currentPage.value = page
}
</script>

<style>
.layout {
  display: flex;
  min-height: 100vh;
  background: #f0f2f5;
}
.content-area {
  margin-left: 240px;
  flex: 1;
  padding: 30rpx 40rpx;
}
</style>
```

**关键点说明：**
- `<component :is="currentComponent" />` 是 Vue 动态组件，根据 `currentPage` 的值切换显示不同的页面组件
- 侧边栏通过 `@navigate` 事件接收菜单点击，`switchPage` 只改变 `currentPage` 的值，不触发页面跳转
- `margin-left: 240px` 与侧边栏宽度一致，保持布局

### 步骤 2：修改侧边栏组件

#### 2.1 修改 `StudentSidebar.vue`

**文件路径：** `uniapp/src/components/StudentSidebar.vue`

**修改 1：在 `<script>` 中添加 `defineEmits`**

```javascript
// 在 props 定义之后添加：
const emit = defineEmits(['navigate'])
```

**修改 2：将导航函数改为 emit 事件**

```javascript
// 修改前：
function goHome() { uni.navigateTo({ url: '/pages/student/home' }) }
function goWrongBook() { uni.navigateTo({ url: '/pages/student/wrongbook' }) }

function goJoinClass() { uni.navigateTo({ url: '/pages/student/join-class' }) }
```

## 五、影响范围与风险

### 5.1 影响范围

| 影响 | 说明 |
|------|------|
| 学生端 | 首页、错题本、成长、加入班级 → 不再独立跳转，改为组件切换 |
| 教师端 | 工作台、上传试卷、新增试题、题库列表、精选、班级管理 → 同上 |
| 教师端子页面 | **不受影响**，保持独立页面和侧边栏 |
| 全屏页面 | **不受影响**，保持 navigateTo 跳转 |
| 管理员 | **不受影响** |
| H5 端 | **不受影响**（H5 本来就已正常工作） |

### 5.2 潜在风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 侧边栏菜单页面组件状态丢失 | 每次切换组件时，之前的组件会被销毁 | 使用 `<keep-alive>` 包裹动态组件可保持状态 |
| 子页面导航回退 | 从 layout 进入子页面后，返回时可能回到错误页面 | 确保子页面使用 `uni.navigateBack()` 返回 |
| 页面数据重新加载 | 组件切换后，新组件会重新触发 `onMounted` | 这是正常行为，与 H5 一致 |
| 当前激活菜单高亮 | 需要确保 `activeItem` 正确传递 | Layout 已通过 `:activeItem="currentPage"` 传递 |

### 5.3 可选优化：使用 `<keep-alive>` 保持组件状态

如果希望切换页面时保留之前页面的滚动位置和数据（不重新触发 `onMounted`），可以将 Layout 中的动态组件用 `<keep-alive>` 包裹：

```vue
<keep-alive>
  <component :is="currentComponent" />
</keep-alive>
```

这样切换回之前的页面时，状态会被保留。但需要注意每个页面都需要有唯一的 `name` 属性（在 `<script>` 中定义），否则 `keep-alive` 无法正确缓存。
